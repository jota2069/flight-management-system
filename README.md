# FlightManagement ✈️
> Production-ready система управления авиарейсами с микросервисной архитектурой

---

## Описание 🌬️

**FlightManagement** — это полноценная серверная система управления авиарейсами (вариант 12), развёрнутая в Docker-контейнерах.  
Система включает REST API, реляционную базу данных, кэширование, мониторинг, CI/CD и веб-клиент с аутентификацией и бронированием билетов.

---

## Основные возможности 👨🏿‍🔬

- Полный CRUD для рейсов, пассажиров, билетов и бортпроводников
- Бронирование билетов через веб-интерфейс с формой пассажира
- Просмотр и отмена билетов в личном кабинете
- Панель администратора: управление рейсами, пассажирами и билетами
- Редактирование рейсов через модальное окно
- Кэширование данных через Redis (TTL 5 минут, инвалидация при изменениях)
- Мониторинг метрик в реальном времени через Grafana
- Автоматический CI/CD пайплайн через GitHub Actions
- Swagger UI с документацией всех 22 эндпоинтов

## Архитектура 🏗️

Система состоит из 7 Docker-контейнеров в единой сети `flight-network`:
Браузер → Nginx (:8080) → API → PostgreSQL
→ Redis
Prometheus → /metrics (API)
Grafana → Prometheus
Веб-клиент (:5001) → API
---

## Интерфейс 🏹

> Табло рейсов (роль: user) — бронирование билетов, просмотр своих билетов, отмена  
> Админ-панель (роль: admin) — управление рейсами, пассажирами и билетами

## Как запустить 🤓

### Требования
- [Docker](https://docs.docker.com/get-docker/) с плагином Compose v2
- Git

### Запуск

1. Клонируйте репозиторий:
```bash
   git clone https://github.com/jota2069/flight-management-system.git
   cd flight-management-system
```

2. Запустите всю систему одной командой:
```bash
   docker compose up --build -d
```

3. Откройте в браузере:

| Сервис | Адрес | Логин |
|---|---|---|
| Веб-клиент | http://localhost:5001 | admin/admin или user/user |
| Swagger UI | http://localhost:8080/swagger | — |
| Grafana | http://localhost:3000 | admin/admin |
| Prometheus | http://localhost:9090 | — |

### Остановка
```bash
docker compose down
```

### Сброс базы данных
```bash
docker compose down -v
docker compose up --build -d
```

## CI/CD ✅

При каждом push в ветку `main` автоматически запускается пайплайн GitHub Actions:

1. Получение кода репозитория
2. Установка .NET SDK 8.0
3. Восстановление зависимостей (`dotnet restore`)
4. Сборка проекта (`dotnet build`)
5. Запуск unit-тестов (`dotnet test`)

Статус последней сборки отображается во вкладке **Actions** репозитория.
